Overview and architecture

Deen Companion uses a hybrid Capacitor shell. The native Android app loads the live web app from Vercel. It does not bundle a local static export.

SettingValue
Package ID (Android)com.deencompanion.lite
Package ID (iOS)com.deencompanion.app (unchanged)
App nameDeen Companion
Remote URLhttps://ramadan-companion.vercel.app
OAuth deep link (Android)com.deencompanion.lite://login-callback
Prayer notifications (native)Local notifications on device; no Firebase or cron required
Hybrid deployment rule Web changes do not appear in the native app until you push to GitHub and Vercel finishes deploying. Verify the live URL before testing native builds.

iOS-only features (expected on Android, not bugs): Sign in with Apple, WidgetKit App Group mirroring via AppDelegate.

Prerequisites

  • macOS recommended (Android Studio also runs on Windows and Linux)
  • Node.js 24 via nvm: project includes .nvmrc and .node-version
  • Android Studio (latest stable) with SDK Platform 36, Build-Tools, and an emulator image (Pixel 6 / API 34+)
  • JDK 21 bundled with Android Studio at /Applications/Android Studio.app/Contents/jbr/Contents/Home
  • Google Play Developer account ($25 one-time): required for Play Store beta upload, not local testing
  • Firebase project — register Android app as com.deencompanion.lite, then add android/app/google-services.json

Install Android Studio

  1. Download from developer.android.com/studio
  2. Open Android Studio → Settings / Preferences → Languages & Frameworks → Android SDK
  3. SDK Platforms tab: install Android API 36 (compile SDK)
  4. SDK Tools tab: ensure Android SDK Build-Tools, Platform-Tools, and Android Emulator are installed
  5. Device Manager → Create Virtual Device → Pixel 6 → API 34 or 35 system image → Finish
JAVA_HOME If Gradle reports "Cannot find Java 21", set JAVA_HOME before building: 
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
Add this to your shell profile (~/.zshrc) for persistence.

Run on emulator

  1. Start your virtual device from Android Studio Device Manager
  2. In the toolbar, select the emulator from the device dropdown
  3. Click Run (green play button) or press Ctrl+R

From the command line:

export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
nvm use 24
npm run cap:run:android

Expected: Splash screen with #0f3d3e theme, then dashboard loads from Vercel.

Run on physical device

  1. On your Android phone: Settings → About → tap Build number 7 times to enable Developer options
  2. Settings → Developer options → enable USB debugging
  3. Connect via USB and accept the "Allow USB debugging?" prompt
  4. Verify: adb devices should list your device
  5. Select the physical device in Android Studio and click Run
Test on real hardware before beta Compass (Qibla), haptics, exact alarm notifications, and widgets behave more reliably on a physical device than on emulators.

Firebase SHA-1 for Google Sign-In

Google OAuth on Android requires signing certificate fingerprints registered in Firebase. Without them, google-services.json has an empty oauth_client array and Google Sign-In fails.

Get debug SHA-1

export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
cd /Users/ahossain/Documents/GitHub/ramadan-companion/android
./gradlew signingReport

Find the Variant: debug section and copy the SHA1 fingerprint.

Alternative using the default debug keystore:

keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android -keypass android

Get release SHA-1 (after creating release keystore)

keytool -list -v \
  -keystore android/keys/deen-companion-release.keystore \
  -alias deen-companion

Add fingerprints to Firebase

  1. Open Firebase Console → project deen-companion-3434b
  2. Project settings → Your apps → Android app (com.deencompanion.lite)
  3. Click Add fingerprint → paste debug SHA-1 → Save
  4. Add release SHA-1 the same way after keystore is created
  5. Download the updated google-services.json
  6. Replace android/app/google-services.json in the repo
  7. Run nvm use 24 && npx cap sync android

Verify Supabase redirect URL

In Supabase Dashboard → Authentication → URL Configuration, confirm this redirect URL is listed:

com.deencompanion.lite://login-callback

See docs/auth-setup.md for full OAuth configuration.

Generate release keystore

Back up your keystore If you lose the keystore or its password, you cannot publish updates to the same Play Store listing. Store copies in a password manager and encrypted backup. 
cd /Users/ahossain/Documents/GitHub/ramadan-companion
mkdir -p android/keys

keytool -genkey -v \
  -keystore android/keys/deen-companion-release.keystore \
  -alias deen-companion \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000

Use a strong password and record it securely. Press Enter at the key password prompt to use the same password as the keystore.

Configure signing

Release signing is wired in android/app/build.gradle and activates when android/key.properties exists. Create that file locally (never commit it):

# android/key.properties
storePassword=YOUR_KEYSTORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=deen-companion
storeFile=../keys/deen-companion-release.keystore

android/key.properties and android/keys/ are listed in .gitignore.

Before each release, increment versionCode in android/app/build.gradle (required by Play Store for every upload).

./gradlew assembleDebug
# Output: android/app/build/outputs/apk/debug/app-debug.apk

Google Play Console setup

  1. Create account at play.google.com/console ($25 one-time fee)
  2. Create app: name Deen Companion, default language English, Free app
  3. Complete required setup tasks:
    • App access: all functionality available without restrictions
    • Ads: No (unless you added ads)
    • Content rating: complete questionnaire (expect "Everyone")
    • Target audience: select appropriate age groups
    • Data safety: declare location and account data per app behavior
    • Privacy policy URL: link to your hosted privacy policy
  4. Store listing assets:
    • App icon 512×512 PNG
    • Feature graphic 1024×500
    • Phone screenshots (minimum 2, recommended 4 to 8)
    • Short description (80 chars) and full description (4000 chars max)

Internal testing track

  1. Play Console → Testing → Internal testing → Create new release
  2. Upload app-release.aab
  3. Add release notes (e.g. "Initial beta: prayer times, Quran, widgets, local notifications")
  4. Review and roll out to internal testing
  5. Testers tab → Create email list → add tester Gmail addresses (up to 100 for internal)
  6. Share the opt-in link; testers accept invite and install from Play Store
Tester install steps Open the opt-in link on Android, accept the invite, then install from Play Store. The app appears as a normal Play Store install, not a sideloaded APK.

Troubleshooting

Gradle sync fails / Java not found

export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"

compileSdk / AAR metadata error (androidx.browser requires API 36)

Ensure android/variables.gradle has compileSdkVersion = 36.

White screen after launch

  • Check device has internet; app loads from Vercel
  • Verify https://ramadan-companion.vercel.app loads in mobile browser
  • Check Logcat in Android Studio for WebView errors

Google Sign-In fails or loops

  • Add debug SHA-1 to Firebase (see Firebase SHA-1 section above)
  • Re-download google-services.json after adding fingerprints
  • Confirm Supabase redirect URL: com.deencompanion.lite://login-callback

Notifications not firing

  • Grant POST_NOTIFICATIONS permission (Android 13+)
  • Disable battery optimization for Deen Companion in system settings
  • On Android 12+, ensure exact alarm permission is granted if prompted
  • Test on physical device; emulators handle alarms inconsistently

Widget shows empty data

  • Open the app at least once after install; widgets read Capacitor Preferences (CapacitorStorage)
  • Set location and wait for prayer times to load, then re-add widget

Resource merge errors

  • res/raw/ filenames must be lowercase (a-z, 0-9, underscore only); no README.md
  • Do not have both splash.xml and splash.png in the same drawable folder

Release build unsigned

Create android/key.properties (see Configure signing section). Without it, release builds compile but are not signed for Play Store upload.

Notification sounds (optional)

To add custom adhaan sounds, place adhaan.wav and silent.wav in android/app/src/main/res/raw/ (lowercase filenames only). Same optional setup exists for iOS in ios/App/App/Sounds/.
